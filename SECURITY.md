# Security Policy

## Reporting a Vulnerability

If you discover a potential security vulnerability in OpenTrusty, please help us keep our users safe by reporting it responsibly. **Do NOT open a public issue.**

Please send security reports via email to: `security@opentrusty.org`

### What we expect from you:
- Detailed steps to reproduce the vulnerability.
- Proof of concept (if applicable).
- A description of the potential impact.

### What you can expect from us:
- Initial response acknowledging the report within 48 hours.
- A proposed timeline for the fix.
- Full credit in our release notes (if desired) once the fix is public.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Responsible Disclosure Policy
OpenTrusty follows a coordinated disclosure model. We ask that researchers allow us a reasonable amount of time (standard 90 days) to deploy a fix before any public information is shared.
